Why dirtrack

Everything you need, nothing you don't

⚡

Blazing fast

Pure Rust + walkdir. Scans 14 million files on an external HDD in under 60 seconds. Sub-second on typical project trees.

🎯

Smart filtering

Built-in presets for secrets, configs, and code. Or pass any comma-separated extensions. Filter by exact filename too.

🧠

Self-teaching UI

Run bare for arrow-key prompts. Every interactive run echoes the equivalent direct command — you learn the flags naturally.

💾

Presets & history

Save with --save daily. Re-run presets or history entries (--run !1). Paths with spaces work — args are stored structurally, not as shell strings.

🗂️

Project-grouped output

Results roll up to top-level projects, sorted by most recently modified. Verbose mode groups the same way with relative file paths.

🔍

Open in file manager

Add --open and pick a project after results. Opens the project root in Finder, Files, or Explorer — no copy-pasting paths.

⚙️

Index cache

Repeat scans skip the directory walk and re-stat known files. 24h TTL with --refresh escape hatch. Dramatic speedup on SSDs.

⌨️

Shell completions

Generate completions for bash, zsh, fish, PowerShell, and elvish with --completions zsh. Tab-complete every flag.

Install in seconds

Requires Rust 1.77+. If you don't have it: rustup.rs (one command).

# Clone and install
git clone https://github.com/deesatzed/mydisasters.git
cd mydisasters
cargo install --path .

# Verify
dirtrack --version

# Shell completions (zsh example)
dirtrack --completions zsh > ~/.zfunc/_dirtrack

Usage

Common patterns

# Secrets changed this week
dirtrack . --since 7d --type secrets
# Config files changed last 2h
dirtrack . --since 2h --type configs -v
# Find any .env, anywhere
dirtrack /var/www --since 30d --file .env
# Save a daily check
dirtrack . --since 1d --save daily
dirtrack --run daily
dirtrack --run !1
# Limit depth (skip vendored deps)
dirtrack . --since 7d --depth 3
# Open result in file manager
dirtrack . --since 2h --open

File type presets

--type in one table
